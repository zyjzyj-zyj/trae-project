# 文档输出

向用户询问关于需求的问题，确认掌握到足够多的信息后，根据用户需求，按照以下顺序生成规范文档：
   
   - requirement.md ：用户视角的需求说明
   - spec.md ：技术规格说明
   - api.md ：API 设计文档（包含每个方法的伪代码）
   - test.md ：测试用例文档

# 案例

  - spec.md 的参考案例文件： assets/spec-example.md
  - requirement.md 的参考案例文件： assets/requirement-example.md
  - api.md 的参考案例文件： assets/api-example.md
  - test.md 的参考案例文件： assets/test-example.md
