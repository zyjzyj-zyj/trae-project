# LLM API 客户端工具 - 需求说明

## 一、用户目标

作为用户，我需要一个简单易用的工具，能够：
- 通过 HTTP 方式调用我指定的 OpenAI 兼容 LLM 服务
- 自动统计每次调用的资源消耗情况
- 快速了解 API 调用的性能表现

## 二、功能需求

### 2.1 配置管理
- 提供配置模板文件 `env.example`，包含必要的连接参数
- 支持用户通过 `.env` 文件自定义配置
- 配置项包括：API 地址、模型名称、访问令牌

### 2.2 API 调用
- 使用 Python 标准库发起 HTTP 请求
- 支持 OpenAI Chat Completions API 格式
- 自动处理请求超时和错误响应

### 2.3 统计功能
- 记录并展示本次调用消耗的 token 数量
- 显示请求的总响应时间
- 计算并展示 token 处理速度（token/s）

### 2.4 用户体验
- 输出清晰的统计信息摘要
- 响应内容易于阅读
- 错误信息友好提示

## 三、使用流程

1. 复制 `env.example` 为 `.env`
2. 编辑 `.env`，填写 LLM 服务配置
3. 运行 `practice01/llm_client.py`
4. 查看统计结果和响应内容

## 四、输入输出示例

### 输入
用户提供的 `.env` 配置：
```
LLM_BASE_URL=https://api.example.com/v1
LLM_MODEL=gpt-3.5-turbo
LLM_API_KEY=sk-xxxxxxxxxx
```

### 输出
程序执行后的控制台输出：
```
========================================
LLM API 调用统计
========================================
模型: gpt-3.5-turbo
提示词 Token: 15
生成 Token: 42
总 Token: 57
响应时间: 2.3s
处理速度: 24.78 token/s
========================================
响应内容:
你好！我是一个 AI 助手。
```

## 五、约束条件

- 仅使用 Python 标准库，无需额外依赖
- 支持 OpenAI 兼容协议的 LLM 服务
- 配置文件必须位于项目根目录
- 响应时间单位为秒，保留两位小数
